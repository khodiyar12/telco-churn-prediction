# Telco Customer Churn Prediction

> End-to-end ML pipeline · IBM Telco dataset · LightGBM + SHAP insights

---

## Project overview

Predict which telecom customers are likely to churn using the [IBM Telco Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) (7,043 customers, 21 features). The project covers the full ML lifecycle: cleaning, EDA, baseline modelling, advanced tree models with Optuna tuning, and SHAP-based explainability to generate actionable retention insights.

**Role:** Data Scientist / ML Engineer Intern  
**Impact:** Directly improves customer retention ROI by flagging at-risk customers before they churn.

## Quickstart

### 1 — Setup

```bash
git clone <your-repo>
cd telco_churn
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2 — Get the dataset

Download from Kaggle and place in `data/`:

```
data/WA_Fn-UseC_-Telco-Customer-Churn.csv
```

### 3 — Run the full pipeline

```bash
python run_all.py
```

Or run each step individually:

```bash
python src/data_cleaning.py        # clean raw CSV
python src/eda.py                  # generate EDA plots
python src/train.py                # train models (with Optuna)
python src/train.py --no-optuna    # train models (fast, default params)
python src/explain.py              # SHAP analysis
python src/predict.py --demo       # predict one synthetic customer
```

## Key findings

| Driver | Insight | Retention action |
|---|---|---|
| **Contract type** | Month-to-month customers churn at 43% vs 3% for 2-year | Offer discounts to lock customers into annual contracts |
| **Tenure** | First 12 months see 47% churn — highest risk window | Proactive onboarding programme in months 1–6 |
| **Payment method** | Electronic check users churn 3× more than auto-pay users | Incentivise bank transfer / auto-pay signup |
| **Tech support** | Customers without tech support churn significantly more | Bundle tech support in entry plans |
| **Monthly charges** | High charges + low tenure = highest risk combination | Price-lock promotions for new high-value customers |
